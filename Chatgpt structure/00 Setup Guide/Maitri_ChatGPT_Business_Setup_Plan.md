# Maitri ChatGPT Business Setup Plan

Last updated: 2026-06-16

## Purpose

This plan explains how to set up Maitri inside ChatGPT Business so Shwethika and Karan can use ChatGPT as an organized working brain instead of a scattered chat archive.

The goal is not to upload everything everywhere.

The goal is to create a few focused Projects, give each Project the right files and instructions, and build a small number of reusable agents for repeated Maitri workflows.

## Current OpenAI Product Reality

Based on OpenAI's current documentation:

- ChatGPT Projects are workspaces for long-running efforts. They can hold chats, files, and project-specific instructions.
- Business users can share Projects with teammates.
- Business Projects currently support up to 40 files per Project.
- Shared Projects use project-only memory, which keeps shared work grounded inside that Project.
- GPTs can use uploaded knowledge files and custom instructions for repeatable behavior.
- Workspace agents are available in research preview for ChatGPT Business and are meant for shared, longer-running workflows that may use tools, approvals, schedules, and connected apps.

Practical interpretation for Maitri:

- Use Projects for context hubs.
- Use GPTs or workspace agents for repeatable jobs.
- Use normal chats for one-off thinking.
- Do not treat one giant Project as the Maitri brain.

## Project Instruction Rules

Every Project should include short instructions that define:

1. What this Project is for.
2. What ChatGPT should prioritize.
3. What it must not do.
4. What questions it should ask before assuming.
5. What format outputs should follow.

Do not put long brand documents into instructions. Upload those as files. Instructions should guide behavior; files should provide knowledge.

## Recommended First Agents

Start with four agents.

### 1. Maitri Chief Of Staff

Best as:

Workspace agent if available. GPT if workspace agents are not enabled yet.

Job:

Organize decisions, summarize founder conversations, turn scattered notes into next steps, and keep Maitri focused.

Use for:

- Meeting notes.
- Weekly priorities.
- Decision logs.
- "What changed?" summaries.
- Shwethika update drafts.

### 2. Manu Story And Product Editor

Best as:

GPT first, workspace agent later.

Job:

Help develop Manu's storybook, letter, activities, sticker ideas, and box experience while keeping the work child-safe and historically careful.

Use for:

- Story outline revisions.
- Page-by-page story planning.
- Activity page ideas.
- Parent-child prompts.
- Tone review.

### 3. Maitri Content Producer

Best as:

GPT first.

Job:

Create social, newsletter, and short-video drafts from approved Maitri strategy and Manu material.

Use for:

- Instagram carousels.
- YouTube Shorts scripts.
- Parent captions.
- Newsletter drafts.
- Content repurposing.

### 4. Maitri Signal Analyst

Best as:

Workspace agent if it will read survey sheets or recurring reports. GPT if analysis is manual for now.

Job:

Convert waitlist, survey, beta-reader, and parent feedback into monthly signal summaries and decisions.

Use for:

- Signup summaries.
- Objection clusters.
- Parent segment analysis.
- Preorder readiness.
- Story and product feedback.

## What Not To Build Yet

Do not build these agents first:

- Full manufacturing agent.
- Legal or compliance agent.
- School-workshop sales agent.
- Multi-character universe generator.
- Automated email sender.
- Public customer-support agent.

Reason:

Maitri is still in foundation mode. These agents will be useful later, but building them now creates more operating surface than the team needs.

## Weekly Operating Rhythm

### Monday

Use Maitri HQ to define the week:

- What matters this week?
- Which decision is blocked?
- What does Shwethika need to review?

### Midweek

Use Product and Content Projects:

- Push Manu box/storybook forward.
- Create content from the approved direction.
- Save strong responses back into the relevant Project.

### Friday

Use Validation Project:

- Review signals.
- Capture objections.
- Decide what changes next week.

## ChatGPT As The Maitri Brain

Think of ChatGPT as four connected but separate brains:

1. Strategy brain: Maitri HQ.
2. Product brain: Manu First Box.
3. Content brain: Content Studio.
4. Evidence brain: Validation And Signals.

This is better than one overloaded brain because each Project stays focused, easier to search, and easier for Shwethika to understand.

## First Setup Checklist

1. Create the four Projects listed above.
2. Upload only the recommended files into each Project.
3. Add the Project instructions from `project_instructions.md`.
4. Share each Project with Shwethika.
5. Create the four agents from `agent_blueprints.md`.
6. Test each agent with one real Maitri task.
7. Save strong outputs back into the right Project sources.
8. Review after one week and remove anything that creates confusion.
