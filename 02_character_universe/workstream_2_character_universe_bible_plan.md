# Workstream 2 - Character Universe Bible Plan

Last updated: 2026-06-27

## Purpose

This plan defines how Maitri should deepen Workstream 2: the Character Universe Bible.

The goal is not to quickly add more heroines. The goal is to build a repeatable character system that keeps Maitri emotionally consistent, historically careful, child-safe, visually coherent, and useful across the website, storybooks, products, content, school material, and future interactive world.

## Strategic Role

The Character Universe Bible should become the source of truth for:

- who each Maitri character is
- what value each character teaches
- what parts are historical, interpreted, or fictionalized
- how each character looks, speaks, behaves, and belongs in the wider Maitri world
- how each character converts into story, product, activities, parent prompts, school use, and content

The immediate priority remains Manu. New characters should be planned only after Manu's bible, story system, and first-box relationship are strong enough to serve as the model.

## Core Approach

### 1. Build Manu as the master pattern

Manu should be treated as the first full prototype for the whole universe.

Before expanding the roster, deepen Manu across:

- emotional arc
- values system
- historical evidence map
- fictionalization boundaries
- child-facing voice
- parent-facing explanation
- visual identity
- product and storybook links
- website and interactive-world use
- content and newsletter hooks

### 2. Separate canon from content

The Character Universe Bible should define canon. Social posts, newsletters, scripts, product copy, and website sections should derive from it.

This prevents Maitri from inventing new personality details each time a piece of content is created.

### 3. Use a layered character model

Every character should be understood through five layers:

1. Historical anchor: the real person, era, geography, and verified facts.
2. Child lens: the age, emotional struggle, question, and everyday courage shown to children.
3. Story world: the setting, objects, relationships, scenes, and recurring motifs.
4. Product expression: doll design, book, accessories, activities, and box experience.
5. Participation layer: parent prompts, school activities, social hooks, newsletters, and interactive-world moments.

### 4. Maintain history with care

Every character must distinguish:

- verified historical fact
- widely repeated but citation-needed claim
- child-friendly interpretation
- invented scene or dialogue
- sensitive or contested material

Maitri can be imaginative, but it should not blur history and fiction.

### 5. Keep courage tender

Maitri's values should avoid turning historical heroines into only warriors, achievers, or moral posters.

Each character should show courage through age-appropriate everyday choices: asking why, trying again, learning, friendship, kindness, patience, curiosity, grief, fairness, and doing what is right.

## Character Bible Structure

Each character bible should eventually include the following sections.

### 1. Character Snapshot

- Name
- Inspired by
- Era
- Region
- Age depiction
- Core value
- Inner question
- One-line character promise
- Child takeaway
- Parent takeaway

### 2. Historical Foundation

- Verified facts
- Source notes
- Claims needing citation
- Contested or sensitive details
- What Maitri will avoid overstating
- What can be safely adapted for children

### 3. Emotional Arc

- What the character wants
- What the character fears
- What feels unfair or difficult
- What brave choice the character makes
- What the character learns
- How the child reader can relate

### 4. Story World

- Main setting
- Secondary settings
- Important people
- Signature object
- Animal, symbol, or recurring motif if relevant
- Festival, food, language, craft, music, or regional cues
- Scene bank for future stories

### 5. Visual Identity

- Color palette
- Clothing direction
- Hair and feature notes
- Expression and posture
- Material and texture references
- Illustration style notes
- Doll design notes
- Safety or production cautions

### 6. Voice And Behavior

- How the character speaks
- Words or phrases they might use
- How they act when afraid
- How they act when curious
- How they treat friends
- What they would not say or do

### 7. Product Expression

- Storybook role
- Doll role
- First-box relationship
- Character card
- Parent note
- Activity sheet
- Accessories and keepsakes
- Future add-on ideas

### 8. Content System

- Character reveal hook
- Parent-facing explanation
- Child-facing prompt
- Instagram carousel ideas
- Reel and YouTube Shorts ideas
- Newsletter themes
- School or workshop activity
- Website module ideas

### 9. Interactive World Role

- Which Maitri Circle location the character belongs to first
- What the child discovers there
- What parents understand there
- Collectible or story object
- Micro-interaction idea
- Guided journey connection

### 10. Review Status

- Draft status
- Historical review status
- Founder review status
- Child-language review status
- Product feasibility status
- Website/content approval status

## Ideas To Explore

### Manu depth ideas

- Build a "Four Courages of Manu" system:
  - asking courage
  - trying courage
  - kindness courage
  - quiet courage
- Create a recurring object system around Badal, the wooden sword, river stones, and letters.
- Turn the Ganga/Bithoor world into the emotional root for the first website and interactive-world story path.
- Develop a parent-child ritual called "What would Manu do today?"
- Create a story-to-play bridge where each chapter ends with a small action a child can do with the doll.

### Universe system ideas

- Give each future character one primary value and one secondary value so the roster does not overlap.
- Build a "Maitri Circle" map where each character enters through a place: Library, Bazaar, Classroom, River Town, Hillside, or future regions.
- Create a shared template for character cards so every doll box feels part of one system.
- Build a "real history / story magic" page pattern for each character.
- Maintain a visible "claim status" for history-sensitive details before they become public copy.

### Website and character-page ideas

- The character page should make Manu feel like a friend first, not a database entry.
- Parent clarity should sit beside child wonder: the page should explain the educational and emotional value without sounding like a school report.
- The site should expose the universe structure gradually: start with Manu, then show that more characters can join when the system is ready.
- Avoid launching a large roster too early. Tease the wider universe through silhouettes, values, places, or future letters rather than unfinished character profiles.

## Instruction Workflow

Use this workflow whenever Maitri creates, expands, edits, or publishes character work.

### Step 1: Source intake

Add raw notes, founder input, references, scans, or emails to `10_source_context/`.

Do not write polished public copy directly from memory or chat discussion without preserving source context.

### Step 2: Historical grounding

For each character, create or update the historical foundation section.

Mark every fact as one of:

- verified
- citation needed
- interpreted
- fictionalized
- avoid for now

### Step 3: Character canon

Update the character bible before creating outward-facing material.

The bible should define identity, emotional arc, setting, visual notes, voice, values, and sensitivity boundaries.

### Step 4: Story and product translation

Translate the bible into storybook, doll, box, accessory, activity, and parent-note ideas.

Keep these in product or story files, but link them back to the character bible.

### Step 5: Content derivation

Create social posts, newsletters, website sections, and scripts only after the character bible is stable enough.

Each piece of content should trace back to one of:

- a verified fact
- an approved fictionalized scene
- a defined emotional lesson
- a product or activity element

### Step 6: Review gates

Before publication, check:

- Is the history separated from fiction?
- Is the language age-appropriate?
- Does it avoid unsafe product or delivery claims?
- Does it keep courage tender and child-safe?
- Does it match Maitri's warm, culturally rooted, parent-trusted tone?
- Has founder review happened where needed?

### Step 7: Status update

Update the character bible review status after changes.

Use simple statuses:

- `draft`
- `needs historical review`
- `needs founder review`
- `approved for internal use`
- `approved for website`
- `approved for product development`

## Maintenance Rules

1. Manu remains the master pattern until the first character system is complete.
2. Do not create deep public pages for new characters before their bibles are strong.
3. Do not publish exact historical claims without citation review.
4. Do not make product safety, materials, manufacturing, preorder, or delivery claims from character files.
5. Keep character work connected to the first-box and website validation strategy.
6. Every public content idea should point back to the character bible.
7. Every character should have a distinct value role; avoid repeating "bravery" in the same way for every heroine.
8. Treat the interactive world as a character-discovery surface, not a separate visual gimmick.
