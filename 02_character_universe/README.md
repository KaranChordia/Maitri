# Maitri Character Universe

This folder contains Maitri's character canon.

Use it to define who each character is before creating website copy, storybooks, social posts, product cards, school material, or interactive-world scenes.

## Source Of Truth

Start here:

1. `workstream_2_character_universe_bible_plan.md` - strategy, approach, ideas, and maintenance workflow.
2. `templates/character_bible_template.md` - reusable structure for every character.
3. `manu/manu_character_bible_v1.md` - current master pattern for the first Maitri character.
4. `manu/manu_story_content_set_v1.md` - content derived from Manu's current canon.

## Working Rule

Canon comes before content.

Do not create public-facing character material until the relevant character bible defines:

- historical anchor
- emotional arc
- value system
- story world
- visual identity
- voice and behavior
- product expression
- content system
- public-use boundaries

## Character Workflow

### 1. Add source context

Place founder notes, research notes, scans, references, or emails in `10_source_context/`.

### 2. Update the character bible

Use `templates/character_bible_template.md`.

Mark facts clearly as:

- verified
- citation needed
- interpreted
- fictionalized
- avoid for now

### 3. Derive content from canon

Only after the bible is stable, create:

- website sections
- story excerpts
- social posts
- newsletters
- activity pages
- school or workshop prompts
- interactive-world scenes

### 4. Run review gates

Before publication, check:

- history is separated from fiction
- child language is warm and age-appropriate
- product, safety, delivery, and preorder claims are avoided unless reviewed
- the tone stays warm, culturally rooted, emotionally intelligent, and parent-trusted
- founder review is complete where needed
